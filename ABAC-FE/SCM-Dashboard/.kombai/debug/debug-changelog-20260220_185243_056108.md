## Changes Made

### 1. Fixed body/root width constraint in global CSS
- **File:** `SCM-Dashboard/src/index.css`
- **Root Cause:** `body { display: flex; place-items: center; }` caused `#root` to take only its intrinsic/content width (≈60–70% of viewport) instead of the full viewport width, constraining all module pages.
- **Change:**
  - Removed `display: flex` and `place-items: center` from `body`
  - Removed `color: rgba(255,255,255,0.87)` and `background-color: #242424` from `:root` (the app manages its own background)
  - Added `width: 100%; padding: 0;` to `body`
  - Added `#root { width: 100%; height: 100vh; }` to ensure full-width rendering
  - Wrapped all global styles in `@layer base` (required by Tailwind v4 to prevent unlayered styles from overriding Tailwind utilities)
- **Revert:** Restore original `index.css` with `body { display: flex; place-items: center; min-height: 100vh; }` and remove `@layer base` wrapper and `#root` rule.
