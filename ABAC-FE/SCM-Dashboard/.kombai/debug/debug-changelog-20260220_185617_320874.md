## Changes Made

### 1. Fixed white button backgrounds in dark mode
- **File:** `SCM-Dashboard/src/index.css`
- **Root Cause:** The global `button` rule in `@layer base` had `background-color: #f9f9f9` (from `@media (prefers-color-scheme: light)`). Buttons that rely on Tailwind's dark-mode context (like inactive nav items, the three-dot menu, and the notification bell) do NOT set an explicit `bg-*` Tailwind class by default, so the `@layer base` white background bled through — even though dark slate backgrounds were applied via parent context or hover states.
- **Change:**
  - Added `background-color: transparent` to the base `button` rule so buttons carry no default fill.
  - Changed `button:hover { border-color: #646cff; }` to `border-color: transparent` — the purple border was leaking on all hovered icon buttons; Tailwind's `hover:bg-*` utilities fully own hover styling.
  - Removed `button { background-color: #f9f9f9; }` from inside `@media (prefers-color-scheme: light)` — redundant with the transparent default.
- **Revert:** Restore the `background-color: #f9f9f9` inside `@media (prefers-color-scheme: light)` button rule, and revert `button:hover { border-color: #646cff; }`.
